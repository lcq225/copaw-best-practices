# 企业微信接入指南

> **版本：** 1.0
> **用途：** CoPaw 接入企业微信的方案对比和配置指南

---

## 一、接入方式对比

企业微信提供两种接入方式：

| 特性 | 智能机器人 | 企业应用 |
|------|------------|----------|
| **配置复杂度** | ⭐ 简单 | ⭐⭐⭐ 复杂 |
| **消息接收** | ✅ 支持 | ✅ 支持 |
| **主动推送** | ❌ 不支持 | ✅ 支持 |
| **群聊支持** | ✅ 支持 | ✅ 支持 |
| **菜单按钮** | ❌ 不支持 | ✅ 支持 |
| **适用场景** | AI 助手、快速接入 | 企业级应用、完整功能 |

---

## 二、智能机器人方案

### 2.1 特点

- **请求-响应模式**：必须先收到用户消息才能回复
- **无法主动推送**：没有 `req_id` 就无法发送
- **跨端不同步**：网页版和企业微信 App 对话记录独立

### 2.2 配置步骤

1. 在企业微信管理后台创建智能机器人
2. 获取 Bot ID 和 Secret
3. 配置 CoPaw 渠道

**配置示例：**
```json
{
    "type": "wecom_smartbot",
    "bot_id": "your-bot-id",
    "secret": "your-bot-secret"
}
```

### 2.3 限制说明

```
用户消息 → 机器人收到 req_id → 机器人可回复
                    │
                    └─ 没有 req_id 就无法发送任何消息
```

**常见场景：**
- ✅ 用户主动发起对话 → 机器人回复
- ❌ 机器人主动推送通知 → 不支持
- ❌ 定时提醒 → 不支持

---

## 三、企业应用方案

### 3.1 特点

- **支持主动推送**：可以向用户发送消息
- **功能完整**：菜单、按钮、卡片等
- **配置复杂**：需要多个凭证

### 3.2 所需凭证

| 凭证 | 说明 |
|------|------|
| CorpID | 企业 ID |
| AgentID | 应用 ID |
| Secret | 应用密钥 |
| Token | 回调验证 Token |
| AESKey | 消息加密 Key |

### 3.3 配置步骤

1. 在企业微信管理后台创建应用
2. 配置回调 URL（需要公网 HTTPS）
3. 获取所有凭证
4. 配置 CoPaw 渠道

**配置示例：**
```json
{
    "type": "wecom_agent",
    "corp_id": "your-corp-id",
    "agent_id": "your-agent-id",
    "secret": "your-agent-secret",
    "token": "your-token",
    "encoding_aes_key": "your-aes-key"
}
```

### 3.4 主动推送示例

```python
# 发送消息给用户
def send_message(user_id, message):
    """
    企业应用可以主动推送消息
    """
    url = f"https://qyapi.weixin.qq.com/cgi-bin/message/send?access_token={get_access_token()}"
    data = {
        "touser": user_id,
        "msgtype": "text",
        "agentid": AGENT_ID,
        "text": {
            "content": message
        }
    }
    requests.post(url, json=data)
```

## 四、如何选择

### 选择智能机器人当

- 只需要用户主动发起对话
- 不需要主动推送通知
- 希望快速配置上线
- 用于 AI 助手场景

### 选择企业应用当

- 需要主动推送消息
- 需要定时提醒功能
- 需要丰富的 UI（菜单、按钮）
- 用于企业级应用场景

---

## 五、混合方案

如果既想快速上线，又需要主动推送：

```
智能机器人（日常对话）+ 企业应用（主动推送）
```

**示例场景：**
- 日常问答 → 智能机器人处理
- 系统告警推送 → 企业应用处理

## 六、常见问题

### Q1: 为什么网页版对话企业微信 App 看不到？

**A:** 智能机器人的两个入口独立，对话记录不同步。

### Q2: 如何实现主动推送？

**A:** 必须使用企业应用方案，智能机器人不支持。

### Q3: 公网地址必须 HTTPS 吗？

**A:** 是的，企业微信要求回调地址必须是 HTTPS。

### Q4: 如何测试企业微信接入？

**A:** 可以使用 ngrok 等工具将本地服务暴露到公网进行测试。

```bash
ngrok http 8000
# 然后将生成的 HTTPS 地址配置到企业微信回调
```

## 七、安全注意事项

1. **凭证安全**：Secret、Token、AESKey 要妥善保管
2. **权限控制**：配置哪些用户可以使用机器人
3. **敏感信息**：不要通过企业微信发送敏感信息
4. **日志审计**：记录关键操作日志
