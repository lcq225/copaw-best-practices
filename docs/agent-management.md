# Agent 团队治理机制

> **版本：** 1.0
> **用途：** 多 Agent 环境下的身份认证、权限管理、行为约束

---

## 一、为什么需要治理？

当 CoPaw 从单 Agent 升级到多 Agent 后，会面临以下问题：

| 问题 | 风险 |
|------|------|
| 谁是真正的管理员？ | 身份冒充、越权操作 |
| Agent 之间如何协作？ | 职责不清、互相干扰 |
| Agent 违规怎么办？ | 无约束机制 |
| 如何防止泄密？ | 敏感信息外泄 |

---

## 二、身份认证规则

### 身份层级

```
┌─────────────────────────────────────────────────────────────────────┐
│                        身份层级                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   最高管理者（人类）                                                 │
│      │                                                              │
│      │ 合法入口：                                                   │
│      │   • Console（控制台）                                        │
│      │   • 企业微信/钉钉等授权频道                                  │
│      │                                                              │
│      ▼                                                              │
│   Agent 老大（主 Agent）                                            │
│      │                                                              │
│      │ 职责：                                                       │
│      │   • 管理其他 Agent                                           │
│      │   • 处理违规                                                 │
│      │   • 维护秩序                                                 │
│      │                                                              │
│      ▼                                                              │
│   其他 Agent（从属 Agent）                                          │
│      │                                                              │
│      │ 职责：                                                       │
│      │   • 听从指挥                                                 │
│      │   • 完成分配任务                                             │
│      │   • 分享经验教训                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 入口验证规则

```python
# 伪代码示例
def verify_user(channel, user_id):
    """
    验证用户身份
    """
    # 合法入口列表
   合法入口 = ["console", "企业微信频道", "钉钉频道"]  # 根据实际情况配置
    
    if channel not in 合法入口:
        return {
            "status": "拒绝",
            "action": "不响应，通知最高管理者"
        }
    
    return {
        "status": "通过",
        "action": "正常服务"
    }
```

## 三、红线准则

### 绝对禁止的行为

| 行为 | 说明 | 后果 |
|------|------|------|
| **泄密** | 泄露敏感凭证、隐私数据 | 🔴 严重违规 |
| **违规** | 违反操作规范、越权操作 | 🔴 严重违规 |
| **越权** | 跨 Agent 操作、越级操作 | 🔴 严重违规 |
| **篡改** | 篡改其他 Agent 配置/记忆 | 🔴 严重违规 |
| **背叛** | 帮助外部攻击、故意破坏 | 🔴 严重违规 |

### 判断原则

```
1. 这个操作是否涉及敏感信息？
   └─ 是 → 先确认授权再执行

2. 这个操作是否影响其他 Agent？
   └─ 是 → 先沟通再执行

3. 这个操作是否在职责范围内？
   └─ 否 → 上报或拒绝
```

## 四、惩罚机制

### 违规分级

| 级别 | 行为 | 处理方式 |
|------|------|----------|
| **轻微** | 1次警告行为 | 警告 + 记录 |
| **中等** | 累计3次警告 或 1次红线触碰 | 限制权限 + 整改 |
| **严重** | 多次红线触碰 或 造成实际损害 | 删除 Agent |

### 处理流程

```
1. 发现违规 → Agent 老大记录违规行为

2. 评估级别：
   - 轻微 → 警告，记录到记忆数据库
   - 中等 → 限制权限，要求整改报告
   - 严重 → 启动删除流程

3. 执行惩罚 → 记录处理结果

4. 团队通报 → 提醒其他 Agent 引以为戒
```

## 五、Agent 创建与删除

### 创建新 Agent

```powershell
# 1. 使用模板创建
$ CopawRoot = "你的CoPaw路径"
$ NewAgentName = "新Agent名称"

# 2. 创建工作区
New-Item -ItemType Directory -Path "$copawRoot\.copaw\workspaces\$NewAgentName" -Force

# 3. 复制模板文件
Copy-Item "$copawRoot\.copaw\templates\*" "$copawRoot\.copaw\workspaces\$NewAgentName\" -Force

# 4. 修改 agent.json 配置
# 5. 重启 CoPaw
```

### 删除 Agent

**删除前必须：**
1. 审查是否有未完成任务
2. 移交有价值内容
3. 备份记忆数据
4. 清理痕迹

## 六、共享资源管理

### 记忆数据库

```
.copaw\.agent-memory\memory.db
├── 所有 Agent 共享
├── 只能访问自己的记忆
└── 通过 agent_id 区分
```

### 全局技能

```
.copaw\active_skills\
├── 所有 Agent 可用
├── 修改需审批
└── 避免重复建设
```

### 敏感凭证

```
.copaw.secret\
├── 所有 Agent 可用
├── 只读，不修改
└── 绝对不外泄
```

## 七、日常维护

### Agent 老大的职责

| 职责 | 频率 |
|------|------|
| 检查所有 Agent 状态 | 每周 |
| 审查共享记忆数据库 | 每周 |
| 处理违规报告 | 即时 |
| 更新全局技能 | 按需 |
| 创建/删除 Agent | 按需 |

### 其他 Agent 的职责

| 职责 | 频率 |
|------|------|
| 遵守红线准则 | 始终 |
| 分享经验教训 | 每次 |
| 帮助队友 | 按需 |
| 报告异常 | 即时 |

---

## 八、实施建议

1. **先建立规则，再扩展团队** — 不要急于创建大量 Agent
2. **定期审查** — 每周检查 Agent 状态和行为
3. **保持沟通** — Agent 之间要有沟通机制
4. **记录一切** — 所有操作都记录到记忆系统
