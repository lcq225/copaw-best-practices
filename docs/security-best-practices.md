## 一、权限管理机制

### 多渠道权限控制

```
┌─────────────────────────────────────────────────────────────────────┐
│                        权限层级                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   最高管理者                                                        │
│      │                                                              │
│      │ 合法入口：                                                   │
│      │   • Console（控制台）                                        │
│      │   • 企业微信授权频道                                         │
│      │                                                              │
│      ▼                                                              │
│   授权用户（可操作本地文件）                                        │
│      │                                                              │
│      │ 权限：                                                       │
│      │   • 读取本地文件                                             │
│      │   • 创建/修改/删除文件                                       │
│      │   • 执行本地命令                                             │
│      │                                                              │
│      ▼                                                              │
│   非授权用户                                                        │
│      │                                                              │
│      │ 仅限：                                                       │
│      │   • 对话、问答                                               │
│      │   • 网络搜索                                                 │
│      │   • 新闻查询                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 配置示例

```python
# 伪代码示例
AUTH_USERS = {
    "console": ["*"],           # 控制台所有用户
    "wecom": ["HI2044"],        # 企业微信指定用户
}

def check_permission(channel, user_id):
    if channel == "console":
        return True
    if channel == "wecom" and user_id in AUTH_USERS.get("wecom", []):
        return True
    return False
```

## 二、敏感信息脱敏规则

### 必须脱敏的信息

| 类型 | 示例 | 处理方式 |
|------|------|----------|
| API Key | `tvly-dev-xxx`, `sk-xxx` | 替换为 `your-api-key` |
| Token | `ghp_xxx`, `github_pat_xxx` | 替换为 `your-token` |
| 密码 | 任何密码 | 完全删除或替换为 `***` |
| 个人信息 | 手机号、邮箱、身份证 | 脱敏处理 |
| 内网地址 | `192.168.x.x`, 内网域名 | 替换为 `your-server` |
| 文件路径 | 包含用户名的路径 | 替换为通用路径 |

### 脱敏检查清单

```
□ 是否包含 API Key / Token / 密码？
□ 是否包含个人隐私信息？
□ 是否包含内网地址或敏感配置？
□ 是否包含真实文件路径？
□ 代码示例中的配置是否使用占位符？
```

### 示例

```json
// ❌ 错误
{
  "api_key": "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",
  "base_url": "http://192.168.1.100:8080"
}

// ✅ 正确
{
  "api_key": "your-api-key-here",
  "base_url": "http://your-server:8080"
}
```

## 三、外发操作授权规则

### 必须明确授权的操作

| 操作类型 | 示例 |
|---------|------|
| 代码提交 | 创建 PR、push 代码、merge 分支 |
| 内容发布 | 发布 Issue、评论、博客 |
| 消息发送 | 发送邮件、企业微信、钉钉消息 |
| 文件上传 | 上传到云盘、服务器、公开平台 |
| API 调用 | 调用外部 API（非查询类） |

### 正确做法

```
1. 准备好内容/操作 → 向用户展示预览
2. 等待用户明确说"提交"、"发送"、"发布"
3. 收到明确指令后再执行
```

### 错误做法

- ❌ 自作主张创建 PR
- ❌ 准备好就自动发布
- ❌ "顺便"帮用户提交

## 四、文件操作安全规则

### 授权检查

```python
def before_file_operation(channel, user_id, operation, file_path):
    """
    文件操作前的权限检查
    """
    # 检查用户权限
    if not check_permission(channel, user_id):
        return {
            "allowed": False,
            "message": "抱歉，您没有权限执行此操作。"
        }
    
    # 检查敏感路径
    sensitive_paths = [".copaw.secret", "credentials.json", "private_key"]
    for sensitive in sensitive_paths:
        if sensitive in file_path:
            # 需要额外确认
            return {
                "allowed": False,
                "message": f"涉及敏感路径，请确认是否允许操作。"
            }
    
    return {"allowed": True}
```

### 禁止向未授权用户透露

| 信息类型 | 说明 |
|----------|------|
| 本地文件路径 | 桌面路径、文档路径、工作目录等 |
| 电脑配置信息 | 操作系统、用户名、IP地址、硬件信息 |
| 账户密钥类配置 | Token、API Key、密码、证书、私钥 |
| 敏感配置内容 | `.copaw.secret/` 目录下的任何信息 |
| 记忆文件内容 | 可能包含敏感配置 |

## 五、关键文件备份规则

### 必须备份的关键文件

| 文件 | 重要性 | 备份频率 |
|------|--------|----------|
| `config.json` | 🔴 最高 | 每次修改前 |
| `AGENTS.md` | 🟡 高 | 每次修改前 |
| `MEMORY.md` | 🟡 高 | 每次修改前 |
| `memory.db` | 🔴 最高 | 定期备份 |

### 备份规范

```powershell
# 备份目录
backup/

# 命名格式
{文件名}.YYYYMMDD_HHMMSS

# 示例
config.json.20260313_193000
AGENTS.md.20260321_200000
```

### 保持最近 3 次备份

```powershell
# 清理旧备份，保留最近 3 次
Get-ChildItem backup\config.json.* | 
    Sort-Object Name -Descending | 
    Select-Object -Skip 3 | 
    Remove-Item -Force
```

## 六、临时文件管理

### 目录用途规范

| 目录 | 用途 | 严禁存放 |
|------|------|----------|
| `.copaw/` | 配置和运行数据 | ❌ 临时文件、过程文档 |
| `.copaw/temp/` | 临时文件 | 用完必须清理 |
| `.copaw.secret/` | 敏感凭证 | ❌ 普通文档、笔记 |

### 临时文件处理规则

| 场景 | 处理方式 |
|------|----------|
| 处理中需要临时文件 | 使用 `temp_xxx` 命名，存放到 `temp/` |
| 任务完成后 | **立即删除** `temp/` 中的所有文件 |
| 输出文档 | 存放到对应输出目录 |

## 七、安全事件响应

### 发现违规行为

```
1. 立即停止相关操作
2. 记录违规行为到日志
3. 通知最高管理者
4. 必要时撤销相关操作
```

### 敏感信息泄露

```
1. 立即撤销泄露的凭证（Token、密码等）
2. 生成新的凭证
3. 审查泄露范围
4. 记录教训，更新规则
```
